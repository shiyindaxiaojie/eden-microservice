# 服务详情页：克制化视觉设计

## 目标

服务详情页以信息层级和留白取代多层卡片与边框。页面保留实例筛选、状态统计、状态切换和分页的现有行为，不改变 API 或权限语义。

## 页面结构

1. 在服务名上方放置紧凑的“返回服务列表”文字链接；不再使用带边框的方形返回按钮。
2. 服务名是唯一的大号主标题。其下将 group 和 namespace 标为含义明确的次级元数据，避免两个无法分辨、颜色不协调的重复标签。
3. 统计数据以一行可点击的数字摘要呈现，用细微状态色区分健康、异常和下线；不放入独立的带框分段容器。
4. “运行实例”区域直接承接页面背景，仅在筛选栏、表头、行和页脚之间保留必要分隔线；移除外围卡片边框和圆角。
5. 刷新保留为轻量按钮。筛选输入框和状态切换保留自身必要边界，作为明确可操作控件。

## 交互与响应式

- 返回链接导航至当前命名空间的服务列表。
- 点击统计摘要与状态筛选保持同步，继续过滤实例列表。
- 窄屏时标题与统计摘要垂直排列；筛选控件换行，状态筛选可横向平分。

## 验证

- 运行 `cd web; npm run check:i18n` 和 `cd web; npm run build`。
- 人工确认：返回不再抢占标题视觉焦点；group、namespace 不重复且可辨；页面不再存在实例区域的外层卡片框。
