# Config 规范

## 1. 定位

Config 是动态配置领域。它以持久化资源的形式保存配置内容，并提供发布、查询、删除、历史、导入导出、md5 版本、监听通知和 Nacos Config HTTP 兼容能力。

Config 不是通用文档库、对象存储、密钥生命周期系统、服务注册模型或网关路由模型。

## 2. 资源身份

Config 使用以下身份：

```text
namespace -> group -> dataId
```

Nacos 兼容参数映射：

| Nacos 参数 | Eden 字段 |
| --- | --- |
| `tenant` | `namespace` |
| `group` | `group` |
| `dataId` | `dataId` |

空 `tenant` 映射为 `default`。空 `group` 映射为 `DEFAULT_GROUP`。

## 3. 设计原则

### 3.1 配置内容是黑盒

Config 只负责保存、版本化、分发和审计完整内容。核心实现不得解析业务字段、合并局部内容或基于某个 YAML/JSON 内部 key 定义核心行为。

### 3.2 md5 表达内容版本

每次发布后根据完整 content 计算 `md5`。客户端监听、Nacos 兼容长轮询和 CAS 发布都以 `md5` 判断内容是否变化。

### 3.3 推送只是提示

变更通知只表示某个配置可能变化。客户端收到通知后必须再次查询配置内容；通知消息不是权威配置内容。

### 3.4 管理面与运行面分离

单配置查询和监听属于运行面。列表、搜索、历史、导入导出、回滚和容量诊断属于管理面，优先给控制台和管理员使用。

## 4. 一期能力

| 能力 | 一期要求 |
| --- | --- |
| 发布 | 创建或更新配置，返回 md5 和 revision |
| 查询 | 根据 namespace/group/dataId 查询内容和元数据 |
| 删除 | 逻辑删除并保留历史 |
| 历史 | 查询最近变更，支持按资源查看 |
| 监听 | 支持精确配置长轮询 |
| 兼容 | 支持 Spring Cloud Alibaba Nacos Config 常用 HTTP 路径 |
| UI | 新增 `配置管理` 菜单 |

灰度发布、标签配置、加密插件和大规模容量治理可以作为后续规范扩展，不进入一期最小闭环。
