# 控制台规范

## 1. 定位

控制台是微服务控制面的管理入口。它负责把注册、配置、路由、命名空间、节点、访问控制和系统设置组织成可操作视图，但不重新定义领域行为。

## 2. 菜单

推荐菜单顺序：

| 菜单 | 路由 | 说明 |
| --- | --- | --- |
| 概览 | `/` | 系统健康、节点、服务、事件和日志摘要 |
| 服务列表 | `/services` | 注册中心服务、实例、拓扑 |
| 配置管理 | `/configs` | 配置中心资源、发布、历史 |
| 路由管理 | `/routes` | 网关路由、上游、过滤器、启停 |
| 命名空间 | `/namespaces` | 命名空间隔离 |
| 节点管理 | `/cluster` | 集群成员和运行状态 |
| 访问控制 | `/rbac` | 用户和角色 |
| 系统设置 | `/settings` | 运行模式、存储、日志、通知、告警 |

`配置管理` 和 `路由管理` 是一级菜单。不要把它们隐藏在服务详情或系统设置里。

## 3. 权限可见性

| 角色 | 配置管理 | 路由管理 |
| --- | --- | --- |
| admin | 读写 | 读写 |
| developer | 读写非系统命名空间 | 读，写权限由后续 RBAC 细化 |
| viewer | 只读 | 只读 |

当前前端只有 admin/developer/viewer 的粗粒度角色时，可以先让 admin 拥有所有写权限，developer 拥有配置写权限和路由只读权限。

## 4. 交互要求

- 配置编辑器应展示 namespace、group、dataId、type、md5、revision 和更新时间。
- 发布配置前显示变更确认；确认弹窗不展示过长完整内容。
- 配置历史可以按 revision 查看和回滚。
- 路由列表应展示启停状态、优先级、匹配条件、上游类型和最近更新时间。
- 路由启停是高风险操作，应有确认。
- 网关数据面状态和控制面保存状态要区分展示。
